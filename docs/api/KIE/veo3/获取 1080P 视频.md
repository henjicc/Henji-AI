# 获取 1080P 视频

获取 Veo3.1 视频生成任务的高清 1080P 版本。

**注意**：通过托底模式生成的视频无法通过此接口访问，因为它们默认已经是 1080p 分辨率。

**端点**
```
GET /api/v1/veo/get-1080p-video
```

**请求示例**
```bash
curl --request GET \
  --url https://api.kie.ai/api/v1/veo/get-1080p-video \
  --header 'Authorization: Bearer <token>'
```

## 使用说明

*   仅 16:9 宽高比的视频支持 1080P 高清生成。
*   视频生成任务成功后，系统会自动开始生成 1080P 高清版本。
*   1080P 视频生成需要额外处理时间，建议在原视频生成完成后等待一段时间再调用本接口。
*   如果 1080P 视频尚未准备好，接口可能返回错误信息。

## 重要说明

*   只有成功生成的视频任务才能获取 1080P 的版本。
*   建议在收到视频生成成功回调后等待几分钟再调用本接口。

## 认证

所有 API 都需要通过 Bearer Token 进行认证。

**Header 参数**

| 参数名 | 位置 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `Authorization` | header | string | 是 | Bearer Token。格式：`Bearer YOUR_API_KEY` |

**获取 API Key**
1.  访问 [API Key 管理页面](https://kie.ai/api-key) 获取您的 API Key。

**注意**
*   请妥善保管您的 API Key，不要与他人分享。
*   如果您怀疑 API Key 已泄露，请立即在管理页面重置。

## 查询参数

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `taskId` | string | 是 | 任务ID |
| `index` | integer | 否 | 视频索引 |

## 响应

**成功响应示例**
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "resultUrl": "https://tempfile.aiquickdraw.com/p/42f4f8facbb040c0ade87c27cb2d5e58_1749711595.mp4"
  }
}
```

**响应字段说明**

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `code` | integer | 响应状态码。 |
| `msg` | string | 响应消息。 |
| `data` | object | 响应数据对象。 |
| `data.resultUrl` | string | 1080P高清视频下载地址。 |

**状态码说明**

| 状态码 | 说明 |
| :--- | :--- |
| 200 | 成功 - 请求已成功处理。 |
| 401 | 未授权 - 认证凭据缺失或无效。 |
| 404 | 未找到 - 请求的资源或端点不存在。 |
| 422 | 验证错误 - 请求参数验证失败。可能原因：记录为空、暂时支持14天内的记录、记录结果数据为空、记录状态不是成功、记录结果数据不存在。 |
| 429 | 请求限制 - 已超过该资源的请求限制。 |
| 451 | 无法获取图片。请验证您或您的服务提供商设置的任何访问限制。 |
| 455 | 服务不可用 - 系统正在进行维护。 |
| 500 | 服务器错误 - 处理请求时发生意外错误。 |
