# 人脸融合

人脸融合 API 用于将图片中的人脸特征融合到模板图片中的人物上。

## 接口信息

*   **请求方法**: POST
*   **请求地址**: `https://api.ppinfra.com/v3/merge-face`

## 请求示例

```bash
curl --request POST \
  --url https://api.ppinfra.com/v3/merge-face \
  --header 'Authorization: <authorization>' \
  --header 'Content-Type: application/json' \
  --data '
{
  "extra": {
    "response_image_type": "<string>"
  },
  "face_image_file": "<string>",
  "image_file": "<string>"
}
'
```

## 响应示例

```json
{
  "image_file": "<string>",
  "image_type": "<string>"
}
```

## 请求头

| 参数 | 类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `Content-Type` | string | 是 | 固定为 `application/json`。 |
| `Authorization` | string | 是 | Bearer 身份验证格式，例如：`Bearer {{API 密钥}}`。 |

## 请求体

| 参数 | 类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `face_image_file` | string | 是 | 包含人脸特征的图片的 base64 编码内容。支持的最大分辨率为 2048 * 2048，文件最大不超过 30 MB。**该图片中必须包含人脸信息。** |
| `image_file` | string | 是 | 模板图片的 base64 编码内容。支持的最大分辨率为 2048 * 2048，文件最大不超过 30 MB。**该图片中必须包含人脸信息。** |
| `extra` | object | 否 | 额外参数。 |
| `extra.response_image_type` | string | 否 | 返回的图片格式，默认为 `png`。<br>可选值：`png`, `webp`, `jpeg` |

## 响应参数

| 参数 | 类型 | 说明 |
| :--- | :--- | :--- |
| `image_file` | string | 融合后图片结果的 Base64 编码内容。 |
| `image_type` | string | 返回图片的格式。<br>可选值：`png`, `webp`, `jpeg` |
